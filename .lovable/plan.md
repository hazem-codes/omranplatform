## Goal
Make the "Submit Your Offer" / "قدم عرضك" experience a fully standalone page with its own route, layout, title, and back button — no longer a dialog nested inside the request details page.

## Current state
Today, on `src/routes/office/requests.$id.tsx`, clicking **Submit Bid** opens a `<Dialog>` (lines 279–309) containing the price/timeline/notes form. It shares the entire request-details layout and component tree.

## Changes

### 1. Create new route `src/routes/office/submit-offer.$id.tsx`
A brand new, self-contained page with:
- Its own `createFileRoute('/office/submit-offer/$id')`
- Its own auth guard (`useAuthGuard('engineering_office')`)
- Its own data fetch: load the request via `supervisorService.getProjectRequestById(id)` so it can show the title/city/budget context at the top of the form (no dependency on the details page)
- Its own page header: title **"قدم عرضك" / "Submit Your Offer"** and its own **Back** button that navigates back to `/office/requests/$id`
- A centered content layout (`mx-auto max-w-xl px-4 py-10`) — visually distinct from the details page's `max-w-[800px]` multi-section layout
- The bid form (price, timeline in days, notes) and submit handler calling `bidService.submitBid(...)` — same logic that currently lives inside `submitBid()` in `requests.$id.tsx`
- On success: toast + navigate back to `/office/requests/$id`
- Loading and error states

### 2. Update `src/routes/office/requests.$id.tsx`
- Remove the entire `<Dialog>` block (lines 279–309)
- Remove the `bidOpen`, `bidForm`, `submittingBid` state and the `submitBid` function
- Remove now-unused imports: `Dialog`, `DialogContent`, `DialogHeader`, `DialogTitle`, `Input`, `Label`, `Textarea` (keep ones still used elsewhere)
- Change the **Submit Bid** button's `onClick` from `setBidOpen(true)` to `navigate({ to: '/office/submit-offer/$id', params: { id: req.request_id } })` so it routes to the new standalone page

### 3. No other files touched
- No changes to `bidService`, navbar, other routes, or styling tokens
- `routeTree.gen.ts` is auto-regenerated by the Vite plugin

## Result
- New URL `/office/submit-offer/<request-id>` renders a completely independent page
- The request details page no longer contains any bid-submission UI or state
- Navigating to "Submit Your Offer" feels like entering a brand new page with its own header, back button, and centered form
